# Домашнє завдання: Django Admin

## Тема проекту
**Онлайн-бібліотека**

Проект не має web-сторінок.
Працюємо **тільки з моделями та Django Admin**.

## Вихідні дані
У проекті є дві моделі:
- автор
- книга

Між моделями існує звʼязок типу:
> один автор -> багато книг

Моделі прості, без складної бізнес-логіки.

## Завдання

### 1. Адмінка для авторів
Налаштуйте сторінку зі списком авторів так, щоб:
- у списку відображалась **корисна інформація про автора**
- список було **зручно переглядати**
- авторів можна було **знайти за текстом**

> Підказка: подумайте, що адміністратор хоче бачити одразу, без відкриття автора.

### 2. Книги всередині автора
Зробіть так, щоб:
- книги автора були доступні **прямо на сторінці автора**
- було зрозуміло, **які книги йому належать**
- адмінка не дозволяла створювати **занадто багато книг одночасно**

> Підказка: редагування має бути зручним, а не перевантаженим.

### 3. Адмінка для книг (список)
Налаштуйте список книг так, щоб:
- у списку була **основна інформація про книгу**
- було **хоча б одне обчислюване поле**
- список можна було **фільтрувати**
- книги можна було **шукати**

> Підказка: уявіть, що книг багато і потрібну треба знайти швидко.

### 4. Адмінка для книг (форма редагування)
Організуйте сторінку редагування книги так, щоб:
- поля були **логічно згруповані**
- форма не виглядала як суцільний список
- другорядні поля не заважали основним

> Підказка: подумайте, які поля описують книгу, а які — її стан або метадані.

### 5. Дії над книгами
Додайте можливість:
- виконувати **дію одразу для кількох книг**
- змінювати їхній стан
- бачити повідомлення про результат виконання

> Підказка: дія має бути корисною саме для бібліотеки.

### 6. Обмеження редагування
Зробіть так, щоб:
- одне з полів книги було **видимим**
- але **не редагувалось вручну**

> Підказка: не всі дані логічно змінювати після створення.

## Додатково (за бажанням)
- зробіть адмінку більш компактною
- покращте читабельність форм
- зменште кількість зайвих елементів на сторінках

## Результат
- проект запускається без помилок
- адмінка працює коректно
- рішення можуть відрізнятись, все залежить від вашої фантазії
- код читабельний та логічний
