# Домашнє завдання: Django REST Framework (DRF)

## Мета

- створення REST API
- серіалізацію даних
- роботу з пов’язаними моделями
- реалізацію CRUD-операцій
- валідацію даних на рівні серіалізаторів

## Вимоги

Потрібно реалізувати REST API для системи **онлайн-курсів**.

У системі є:
- курси
- викладачі
- студенти

### Логіка

- Один курс має **одного викладача**
- Один курс може мати **декілька студентів**
- Один студент може бути записаний на **декілька курсів**

## Моделі

Необхідно створити такі Django-моделі:

### Instructor (Викладач)
- імʼя
- email
- досвід роботи (у роках)

### Student (Студент)
- імʼя
- email
- дата реєстрації

### Course (Курс)
- назва курсу
- опис
- дата старту
- викладач
- список студентів

Типи полів і звʼязків між моделями потрібно визначити самостійно, виходячи з логіки задачі.

## Серіалізатори

Створити серіалізатори з використанням **ModelSerializer**.

### Для Course:
- у відповіді API повинні повертатися **вкладені дані** про викладача та студентів
- серіалізатор має підтримувати:
  - створення курсу
  - оновлення курсу
  - зміну списку студентів

Для роботи зі вкладеними обʼєктами необхідно реалізувати власну логіку в методах:
- `create`
- `update`

## Валідація даних

У серіалізаторах необхідно реалізувати **валідацію даних**.

### Вимоги до валідації:
- дата старту курсу не може бути в минулому
- досвід викладача не може бути відʼємним
- email повинен бути у коректному форматі

### Додатково:
Реалізувати **хоча б одну кастомну валідацію** на власний розсуд  
(наприклад: мінімальна кількість студентів на курсі, довжина назви курсу тощо).

> Підказка:  
> Для валідації можна використовувати в серіалізаторах:
> - метод `validate(self, attrs)`
> - методи виду `validate_<field_name>(self, value)`

## APIView

Необхідно реалізувати API для роботи з курсами.

### Курси:
- отримання списку курсів (get)
- створення курсу (post)
- перегляд деталей курсу (get)
- повне оновлення курсу (put)
- часткове оновлення курсу (patch)
- видалення курсу (delete)

APIView має працювати з JSON та повертати коректні HTTP-статуси.

## Технічні вимоги

- використовувати **APIView**
- використовувати **ModelSerializer**
- DRF має бути налаштований глобально через `settings.py`
- можна використовувати **Browsable API** для тестування

## Результат

- Django-проєкт з коректно працюючим REST API
- правильна серіалізація повʼязаних моделей
- реалізована валідація даних
- можливість тестування API через браузер

## Додатково (необовʼязково)

- додати сортування курсів
- додати фільтрацію
- додати зрозумілий `__str__` для моделей
